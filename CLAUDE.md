# Claude Code Guidelines for Aiki

## Architecture Decision Process

Before proposing any new structure, file location, naming, or pattern:

1. **Find similar examples** - Search the codebase for 2-3 similar existing implementations
2. **Study the patterns** - Examine how they're organized, named, and structured
3. **Follow exactly** - Match the existing conventions; don't invent new patterns

**Read before write, copy before create.**

This applies to:
- File and directory locations
- Naming conventions (match existing API names, e.g., `hasTerminated` mirrors `hasTerminatedV1`)
- Code structure and organization
- Documentation format and placement
- Example file structure
